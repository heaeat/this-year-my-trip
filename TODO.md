# 나중에 수정할 항목

## 🗺️ 가고싶은 곳 — 바이두맵 링크 감지 오류
**증상**: 중국 여행에도 바이두맵 버튼이 안 뜨는 경우 발생
**원인**: `isChina` 판단을 trip 이름의 한자 포함 여부로만 체크 → 여행 이름을 한글로 써도 중국인 경우 감지 못 함
**현재 코드** (`rTabWishlist`):
```js
const isChina=/[\u4e00-\u9fff]/.test(trip.name);
```
**개선 방향**:
- 여행에 "국가" 또는 "지역" 필드를 별도로 추가 (예: `trip.country = 'CN' | 'KR' | ...`)
- 또는 여행 생성/수정 모달에서 중국 여행 체크박스 추가
- 저장된 장소의 좌표가 중국 영역(lat 18~53, lng 73~135)인지로 자동 판단하는 방법도 있음

## 📋 전체보기 개선
**증상**: 전체보기 화면이 직관적이지 않다고 느껴짐
**현재 상태**: 카테고리별 요약 카드 방식으로 1차 개선했으나 아직 아쉬운 부분 있음
**개선 방향**: 추후 논의 후 결정

---

## 🏷️ 앱 이름 & 로그인 화면 멘트 변경
**현재**: 앱 이름 "올해의 여행", 멘트 "친구들과의 여행 예약을 한눈에 관리하세요"
**위치**: `rLogin()` 함수 내 `<h1>`, `<p>` 태그
**후보 이름**:
- 여정 (旅程)
- 길위에서
- 떠나자
- 오늘도 어딘가로
- Wandr / Triply / somewhere
- 지도 밖으로 / 다음 목적지

---

## 🚗 렌터카 탭 — 예약 삭제 시 탭 자동 전환 없음
**증상**: 렌터카 탭이 활성화된 상태에서 렌터카 예약을 모두 삭제하면 탭이 사라지지만 `S.activeTab`은 `'cars'`로 남아 빈 화면 표시
**개선 방향**: `delR()` 또는 렌더링 시점에 렌터카 예약이 없는데 `activeTab==='cars'`이면 `'budget'`으로 자동 전환

---

## 🔄 실시간 데이터 연동 (onSnapshot)
**증상**: 친구가 정산완료 등 변경해도 내 화면에 실시간 반영 안 됨 (새로고침 필요)
**현재**: `load()` 함수에서 `get()` (일회성 읽기) 사용
**개선 방향**: Firestore `onSnapshot` 리스너로 변경하면 실시간 양방향 동기화 가능. 변경 범위가 크므로 별도 작업 권장.

---

## 📝 기타 메모
- 바이두 URL 좌표 변환: URL의 `@x,y,z` 좌표가 POI 중심점이 아닌 뷰포트 중심일 수 있어 약간 오차 있음 (해결 어려움, 현상 유지)
- 구글맵 API 할당량 제한 설정 필요 (Google Cloud Console → Places API → 할당량 → 하루 100건)
